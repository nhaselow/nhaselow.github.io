NBA Web Data Scraper

Spring 2017

Every data analyst needs data, and my job was to get it to them!

Built With:   Python,  BeautifulSoup4

What I Learned (so far): Efficiency matters! Instead of iterating through each possible url (14,000+ possibilities taking over 4 hours), the scraper iterates through a page of summaries of games for each month, determining valid urls, reducing the scraper's running time by over 90% from its first build.

About the Project: In order to get their name out, the newly founded UW Sports Analytics Club decided to attend the 2017 MadHacks competition and built a regression model predicting NBA regular season game scores. My primary job on this team was to build a web scraper to retrieve data from basketball-reference.com and export it into a format that made it easy for the statisticians on our team to manipulate in R.

I used BeautifulSoup's web scraping module to scrape through the 1200+ box scores from 2014-2016 to get 75+ stats per game. This list was then converted into a .csv file formatted such that it may be easily converted to a data frame in R.

What's Next? I am currently working with a team of developers to develop an easy to use UI for statisticians which allows them to select a specific time period of data to scrape. We are also working on automating the scraper to get up-to-date game stats daily to continuously test the regression model.
